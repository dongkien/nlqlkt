Lab 0 — Cài Stata và các lệnh cơ bản

Hướng dẫn nền tảng trước khi vào 15 lab buổi

Note🎯 Mục tiêu Lab 0

Sau Lab 0, bạn có thể:

  • Mở Stata trên máy phòng FTU hoặc máy cá nhân
  • Tải, đọc, ghi dataset (.dta, .csv, .xlsx)
  • Khám phá nhanh một dataset bằng describe, summarize, tabulate
  • Viết do-file đầu tiên có comment tiếng Việt
  • Lưu kết quả ra log file

1. Tiếp cận Stata

Phương án A — Phòng máy FTU (khuyên dùng cho sinh viên)

Đa số phòng máy của FTU có Stata cài sẵn. Lịch sử dụng:

  • Hỏi giảng viên về lịch phòng máy dành cho lớp DTU301
  • Mang theo USB để lưu file do/log/dataset của bạn
  • Mỗi buổi lên phòng máy 1-2 lần là đủ làm xong 1-2 lab

Phương án B — Mua Stata BE cho sinh viên (~$129/năm)

Nếu bạn dự định làm khoá luận hoặc tiếp tục nghiên cứu lâu dài, cân nhắc mua Stata BE student:

  • Đăng ký trên stata.com — cần ảnh chụp thẻ sinh viên FTU
  • Cài đặt trên Windows / macOS — quy trình tự động
  • Có quyền sử dụng tới khi tốt nghiệp

Phương án C — Stata Cloud (trải nghiệm ngắn)

Stata có trial cloud thông qua trình duyệt — không cài đặt:

  • Vào https://www.stata.com/
  • Có thể trial trong giới hạn nhất định
  • Không lưu được do-file dài hạn

2. Cấu trúc cửa sổ Stata

Khi mở Stata lần đầu, có 5 cửa sổ chính bạn cần biết:

Cửa sổ Vai trò
Command (dưới đáy) Gõ lệnh ngay — chạy 1 dòng tức thì
Results (giữa) Output của mọi lệnh đã chạy
Variables (bên phải) Danh sách biến trong dataset hiện tại
Properties (góc phải dưới) Thông tin về biến đang chọn
History (bên trái) Lịch sử lệnh đã chạy

Cửa sổ thứ 6 — Do-file Editor — mở bằng nút “Do-file Editor” hoặc Ctrl+9. Đây là chỗ bạn viết script thay vì gõ từng lệnh.

Tip🔑 Khẩu lệnh quan trọng nhất

Đừng gõ lệnh ở Command window! Luôn dùng Do-file Editor. Lý do: bạn cần tái tạo lại được phân tích về sau, và cần nộp file .do cho giảng viên.

3. Viết do-file đầu tiên

Mở Do-file Editor (Ctrl+9) và gõ:

* Lab 0 — DTU301 — Buổi học đầu tiên với Stata
* Họ tên: <điền tên>
* MSSV: <điền MSSV>
* Ngày: <điền ngày>

clear all                    // Xóa dữ liệu cũ trong bộ nhớ
set more off                 // Không phải bấm space để xem tiếp output

* Tải dataset có sẵn của Stata
sysuse auto, clear

* In thông báo ra màn hình
display "Stata đang chạy. Bạn đã mở dataset auto.dta"

* Xem cấu trúc dataset
describe

* Xem 5 dòng đầu
list in 1/5

Lưu file với tên lab00_<tên>.do ở thư mục bạn chọn.

Chạy file: nhấn nút Execute (do) — hoặc phím tắt Ctrl+D để chạy tất cả, Ctrl+R để chạy phần được bôi đen.

4. Khám phá dataset — bốn lệnh cốt lõi

describe — mô tả cấu trúc

describe

Output sẽ cho biết: - Số quan sát (observations) — bao nhiêu dòng - Số biến (variables) — bao nhiêu cột - Kiểu dữ liệu từng biến (byte, int, float, str…) - Label của biến (nếu có)

summarize — thống kê mô tả số

summarize
summarize price mpg weight
summarize, detail   // có thêm phân vị, độ lệch, kurtosis

Đây là lệnh thường dùng nhất. Hãy luôn summarize ngay sau khi mở dataset mới.

tabulate — bảng tần suất biến phân loại

tabulate foreign
tabulate foreign, summarize(price)   // có cả mean price theo nhóm
tabulate foreign rep78               // bảng chéo

list — xem dòng dữ liệu thô

list make price mpg in 1/10            // 10 dòng đầu
list if mpg > 30                       // tất cả dòng có mpg > 30
list make price if foreign == 1        // xe nhập khẩu
Tip📝 Bài tập nhỏ ngay tại Lab 0

Mở dataset auto, trả lời các câu hỏi:

  1. Có bao nhiêu xe trong dataset?
  2. Giá trung bình là bao nhiêu?
  3. Có bao nhiêu xe nội địa (foreign == 0) và bao nhiêu xe nhập khẩu (foreign == 1)?
  4. Xe nào có mpg cao nhất?
  5. So sánh giá trung bình giữa xe nội địa và xe nhập khẩu — xe nào đắt hơn?

5. Tạo biến mới và lọc dữ liệu

generate — tạo biến mới

* Tạo biến mới từ công thức
generate price_thousand = price / 1000
generate is_expensive = (price > 10000)
generate ln_price = log(price)

replace — sửa giá trị biến hiện có

replace mpg = 25 if make == "Honda Accord"

keep / drop — giữ / xóa biến hoặc dòng

keep make price mpg foreign          // Giữ 4 biến này, bỏ phần còn lại
drop if price < 4000                  // Xóa các dòng có price < 4000
keep if foreign == 1                  // Giữ chỉ xe nhập khẩu

sort — sắp xếp

sort price                            // tăng dần theo price
gsort -price                          // GIẢM dần theo price

6. Vẽ biểu đồ cơ bản

Scatter plot

twoway scatter price mpg
twoway scatter price mpg, title("Giá vs Tiêu thụ xăng") ///
    xtitle("Mile per gallon") ytitle("Giá (USD)")

Bar chart

graph bar (mean) price, over(foreign)
graph bar (mean) price (mean) mpg, over(foreign)

Histogram

histogram price
histogram price, by(foreign) normal

7. Lưu output ra log file

Để chấm Stata Lab, bạn cần nộp file .log chứng minh code đã chạy. Cách bật log:

* Đầu do-file
log using "lab00_<tên>.log", replace text

* ... toàn bộ phân tích của bạn ở đây ...

* Cuối do-file
log close

File log sẽ ở cùng thư mục với do-file.

8. Lưu và đọc dataset

Lưu dataset hiện tại

save "mydata.dta", replace          // Lưu định dạng Stata
export delimited "mydata.csv", replace   // Lưu CSV

Đọc dataset bên ngoài

use "mydata.dta", clear             // Đọc .dta
import delimited "mydata.csv", clear case(lower)   // Đọc CSV
import excel "mydata.xlsx", firstrow clear         // Đọc Excel

9. Mẹo làm Stata hiệu quả

  1. Luôn dùng do-file, không gõ lệnh tay
  2. Comment tiếng Việt sau mỗi đoạn code
  3. Đặt tên biến dễ hiểu: gdp_thuc thay vì g1
  4. Sử dụng /// để xuống dòng trong 1 lệnh dài
  5. Bật set more off đầu file để không phải bấm space
  6. Save thường xuyên dataset đã làm sạch
  7. Tạo working directory chuyên cho DTU301: cd "đường-dẫn" đầu file

10. Lệnh giúp đỡ

Khi không nhớ syntax:

help summarize         // Hướng dẫn chi tiết
help twoway            // Hướng dẫn về biểu đồ
search PPF             // Tìm tài liệu, ví dụ
