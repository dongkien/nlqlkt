# Checklist Pilot — Giảng viên

Dành cho **giảng viên** (TS. Đỗ Ngọc Kiên) khi triển khai pilot test Lab Stata.

## 0. Chuẩn bị (1 tuần trước)

- [ ] Chọn **5–8 sinh viên** tự nguyện làm pilot (đa dạng học lực, không chỉ giỏi)
- [ ] Gửi **gói tài liệu** trong `/student-package/`:
  - [ ] `HUONG-DAN-PILOT.md`
  - [ ] `lab-00.html`
  - [ ] `lab-01.html`
  - [ ] `PHAN-HOI.md` (form thu thập)
- [ ] Xác nhận với phòng máy FTU rằng Stata sẵn sàng
- [ ] Tạo địa chỉ email/Drive nhận bài

## 1. Khi sinh viên làm (5–7 ngày)

- [ ] **Trả lời thắc mắc nhanh** (24h) — qua group chat hoặc email
- [ ] **Không can thiệp** quá trình làm — để sinh viên tự gặp lỗi (đây là dữ liệu quý)
- [ ] **Ghi nhận** mọi lỗi sinh viên báo

## 2. Sau khi nhận bài (1 tuần)

### Phân tích định lượng

| Câu hỏi | Cách phân tích |
|---|---|
| Thời gian thực tế vs hứa hẹn | So sánh trung bình |
| % sinh viên hoàn thành Lab 0 | Đếm |
| % sinh viên hoàn thành Lab 1 | Đếm |
| Tỷ lệ "code không chạy" | Theo dõi lỗi cụ thể |
| Tỷ lệ "rất tốt" trong B.5 | Phần trăm |

### Phân tích định tính

- [ ] Đọc kỹ **3 câu hỏi mở B.1–B.4**
- [ ] Tìm **pattern** chung trong lỗi gặp phải
- [ ] Ghi nhận **đề xuất cải tiến** cụ thể nhất

## 3. Tiêu chí "Lab đạt" (theo định tính)

| Tiêu chí | Ngưỡng đạt |
|---|---|
| Code chạy đúng | ≥ 80% sinh viên báo OK |
| Thời gian thực vs hứa hẹn | ± 30% (vd hứa 30 phút → 20-40 phút) |
| Liên kết lý thuyết tốt | ≥ 60% chọn "Tốt" hoặc "Rất tốt" |
| Có hứng thú làm tiếp | ≥ 50% chọn "Có" hoặc "Có nếu bonus" |
| Bonus 5% đủ hấp dẫn | ≥ 50% chọn "Vừa" hoặc "Rất" hấp dẫn |

## 4. Hành động sau pilot

### Nếu **Lab đạt**:

- [ ] Cập nhật Lab dựa trên góp ý
- [ ] Mở rộng pilot lên 20–30 sinh viên (lớp đầy đủ)
- [ ] Chuẩn bị triển khai chính thức kỳ sau

### Nếu **Lab chưa đạt**:

- [ ] Xác định *nguyên nhân chính*:
  - Quá khó? → giảm độ phức tạp
  - Code có bug? → sửa
  - Liên kết yếu? → nhấn mạnh trong bài giảng
  - Không hấp dẫn? → tăng bonus hoặc nội dung *thú vị* hơn

## 5. Báo cáo Pilot

Sau khi phân tích, viết báo cáo ngắn (2–3 trang):

1. **Tóm tắt** số liệu định lượng
2. **Bài học chính** từ phản hồi
3. **Đề xuất sửa đổi** cụ thể cho từng Lab
4. **Kế hoạch** triển khai chính thức

Báo cáo này có thể dùng làm **case study** cho bài tham luận giáo dục, hoặc làm cơ sở để Bộ môn duyệt chính thức Lab vào đề cương.
